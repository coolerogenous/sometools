# 加密货币项目调查员（Crypto Project Researcher）

## 技能概述

本技能帮助用户全面调查加密货币项目，评估其作为撸毛（空投）机会的潜力。通过整合多个数据源（官网、社交媒体、GitHub、数据平台等），提供详细的项目分析报告。

## 核心能力

### 1. 多源数据收集
- 访问项目官网和文档
- 查询社交媒体数据（X/Twitter, Discord）
- 分析 GitHub 开发活跃度
- 检查主流数据平台（CoinMarketCap, CoinGecko, DeFiLlama）
- 搜索融资和团队信息

### 2. 项目评估维度
- **团队背景**: 创始人、顾问、技术实力
- **融资情况**: 投资轮次、投资机构、融资金额
- **TVL 和用户数据**: 总锁仓量、活跃用户数
- **生态系统**: 合作伙伴、集成项目、社区活跃度
- **技术实力**: 开发活跃度、代码质量、创新性
- **代币经济学**: 代币分配、空投计划、激励机制

### 3. 撸毛机会评估
- 空投可能性（高/中/低）
- 交互成本（Gas 费、时间成本、技术门槛）
- 预期收益（空投价值、代币潜力）
- 风险评估（项目风险、女巫检测风险）

## 工作流程

### 标准调查流程

```
1. 确认项目名称和目标
   ↓
2. 访问项目官网和文档
   ↓
3. 查询社交媒体数据
   ↓
4. 分析 GitHub 开发活跃度
   ↓
5. 检查主流数据平台
   ↓
6. 搜索融资和团队信息
   ↓
7. 评估撸毛机会
   ↓
8. 生成综合分析报告
```

### 详细步骤

#### 步骤 1：项目确认
- 确认项目名称
- 了解调查目的（撸毛、投资、了解项目）
- 确定关注重点（技术、团队、生态等）

#### 步骤 2：官网和文档分析
使用 Chrome DevTools MCP 访问：
- 项目官网
- 官方文档
- 白皮书/技术文档
- API 文档（如有）

**关键信息提取**：
- 项目定位和愿景
- 核心产品和服务
- 团队和顾问信息
- 合作伙伴和生态项目
- 代币信息（如有）

#### 步骤 3：社交媒体分析
访问：
- X (Twitter): https://x.com/<项目账号>
- Discord（如有）
- Telegram（如有）

**关键指标**：
- 关注者数量
- 帖子数量和互动量
- 社区活跃度
- 项目时间线（启动时间、重要里程碑）

#### 步骤 4：GitHub 分析
访问：https://github.com/<组织名>

**关键指标**：
- 仓库数量
- Stars 和 Forks 数量
- 最近提交时间
- 主要编程语言
- 开发活跃度

#### 步骤 5：数据平台查询
访问：
- CoinMarketCap: https://coinmarketcap.com/currencies/<项目>
- CoinGecko: https://www.coingecko.com/en/coins/<项目>
- DeFiLlama: https://defillama.com/protocol/<项目>
- Messari: https://messari.io/profile/<项目>

**关键信息**：
- 是否被收录
- TVL 数据
- 交易量
- 市值（如有代币）

#### 步骤 6：融资和团队搜索
使用 WebSearch 查询：
- 项目融资信息
- 投资机构
- 团队成员背景
- 顾问团队

#### 步骤 7：撸毛机会评估
基于收集的信息，评估：

**空投可能性**（高/中/低）：
- 是否有代币计划
- 是否有积分系统
- 是否有空投公告
- 历史空投记录

**交互成本**：
- Gas 费用
- 时间成本
- 技术门槛
- 资金要求

**预期收益**：
- 空投价值预估
- 代币潜力
- 市场热度

**风险评估**：
- 项目风险（技术、市场、团队）
- 女巫检测风险
- 资金安全风险

#### 步骤 8：生成报告
生成包含以下部分的 Markdown 报告：

- 项目基本信息
- 团队和顾问
- 融资情况
- TVL 和用户数据
- 生态系统
- 代币和空投情况
- 撸毛机会分析
- 最终结论和建议

## 评分标准

### 撸毛机会评分（1-5星）

| 评分 | 标准 |
|------|------|
| ⭐⭐⭐⭐⭐ | 极高：有明确空投计划，TVL适中，团队强，成本低 |
| ⭐⭐⭐⭐ | 高：可能有空投，TVL中等，团队不错 |
| ⭐⭐⭐ | 中等：不确定是否有空投，TVL较小或较大 |
| ⭐⭐ | 低：不太可能有空投，TVL很小或成本高 |
| ⭐ | 极低：没有代币计划，TVL极小，成本极高 |

### 项目成熟度评分（1-5星）

| 评分 | 标准 |
|------|------|
| ⭐⭐⭐⭐⭐ | 成熟：TVL大，代币已上线，生态完善 |
| ⭐⭐⭐⭐ | 较成熟：TVL中等，代币上线，生态发展 |
| ⭐⭐⭐ | 中期：TVL较小，代币未上线，生态建设中 |
| ⭐⭐ | 早期：TVL很小，无代币，生态初期 |
| ⭐ | 极早期：TVL极小，无代币，无生态 |

## 最佳实践

### 1. 数据验证
- 交叉验证多个数据源
- 检查数据更新时间
- 注意数据平台的收录延迟

### 2. 风险控制
- 始终提醒用户风险
- 不要给出投资建议
- 强调 DYOR（Do Your Own Research）

### 3. 报告结构
- 使用清晰的标题结构
- 添加表格对比
- 使用表情符号增强可读性
- 提供明确的结论和建议

### 4. 语言风格
- 客观中立
- 数据驱动
- 避免过度乐观或悲观
- 使用专业术语

## 限制和注意事项

### 技术限制
- 某些网站可能有 Cloudflare 验证
- 需要登录的页面无法访问
- 动态加载内容可能需要等待
- 反爬虫机制可能限制访问

### 内容限制
- 无法访问需要登录的内容
- 无法绕过付费墙
- 某些地区可能有访问限制
- 实时数据可能不准确

### 性能限制
- 大型项目分析可能需要较长时间
- 深度分析会消耗更多资源
- 多页面分析需要合理规划

## 故障排除

### 常见问题

**问题 1：无法打开网页**
- 检查 URL 是否正确
- 确认网络连接
- 检查网站是否可访问

**问题 2：数据平台未收录项目**
- 项目可能太早期
- 检查项目名称拼写
- 尝试其他数据源

**问题 3：GitHub 数据不完整**
- 组织可能是私有的
- 检查是否有其他相关仓库
- 搜索团队个人账户

**问题 4：社交媒体数据异常**
- 检查账号是否正确
- 确认账号是否被封禁
- 尝试其他社交平台
